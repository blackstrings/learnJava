Fence

Fence is one of our legacy c++ applications. It is also one of the simplest application on our kiosks. It allows a guest to create a basic line drawing of what their fence layout will look like and create an estimate based on the products they selected. There are a wide range of choices to choose from.

Special Notes

  • Fence uses INI files for managing screen and telling the application what we want to see on the screen (btns, text, etc). In debug you will see a number ex: 200 on top left, so the file to manipulate that screen is in Step200.INI
  • Can't access INI files inside VS6, use window browsers instead
  • Adding/Deleting buttons are dynamic. If you increase the number of buttons on a page which exceeds the screen space availability, a 2nd screen will be generated automatically

Terminology

  • SCodes = like SKU they are used to map to SKU numbers for easy updating purposes
  • INI files : use for displaying media on screen

Setup

  • Visual Studio 6 (IDE) (C++)

Side Stuff

  • Preveious - go back
  • Textbanner - to display any text you see (except button text)
  • Each text you add, you have to also update the total count
  • Two configuration: Release & Debug - we work in debug
