Code Collaborator (CC)

Though we are no longer using Code Collaborator, it has been replaced with Fisheye + Crucible, code collaborator is a third-party online site we use for sharing our source codes that we have modified, changed, deleted, or added. Once uploaded to CC, other team members are then assigned to review and observe the code changes. This is done so that codes can be properly checked for accuracy before they get merged/incorporated into the latest app version.

Highlights

  • You'll need a login to access CC
  • http://codereview.menards.net/
  • We create a code review for each MCR ticket in CC and assign them to the primary and secondary to review to approve and then the rest of the team members are observer. We always add Merica (team lead) as an observer.

Process

  1. After Redmine construction phase, which is after you have went on to make the code changes on a branch and have SVN commit the branch back to the feature repository folder, you want to create a code review in CC for the codes you have changed.
  2. During creating a code review, you will attach the feature branch and CC will somehow upload the correct branch from SVN to CC for you.
  3. Assign the primary and secondary and an observer.

Creating a Review

  • After you commit your feature branch to the Repo using tortoiseSVN
  • CC browser > create new review > Title: same as MCR > Overview: summary of what you did > Set Author, Reviewers, Observers
  • Attach Checked-in Changelist

Fix revision after defect is found

  • Go and fix the code where TM founded defects.
  • In window explorer, highlight and SVN commit the feature branch back into the repo
  • Go to CC browser > Click on your review > Edit > scroll down > click attach checked-in changelist
  • show log on feature branch in SVN repo to get the log window below on right
  • Only put (one) the latest revision number into the changelist > continue
  • Note that CC cares only about the revision number and is how CC tracks for commit changes
