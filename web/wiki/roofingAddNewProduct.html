Adding new products to Roofing

There are multiple property places that need to be in place for a main roofing style shingle products.

Adding new shingle style

Add the style

This is where you define how much coverage one bundle is or how much 1 sheet coverage is, Use square inch per bundle for bundles. If you are given feet, convert it to square inch. Use lengthperpeice if style doesn't come in bundle. In this case, pieceperbundle will be 1 and the length is filled.

tiles-def.xml

Add the new style onto the screen. In tiles-def jump to screen 1502, add the style and create a putlist of its shingle colors. See add shingle below to get the Scodes for the shingles of that style.

Add the main image

Create image ticket to request image 216x153 size and add it to
/media/images/shingleRoofingType/ Name the image properly, ex: the first 7 characters shuld be filled followed by underscore = BRIARWD_XX.jpg

Adding new shingle to existing style

  1. Give the sku a new Scode in roofingSkuID.properties
  2. Start defining its values in tsShingles.properties (ts = traditionalShingle)

  3. Further down tsShingles.properties, define the property values. You'll have to also create new scode for hipRidge for each color. As every shingle color has a specific hip ridge color (half true).

    Each shingle sku has 4 basic properties as you see below.
    in pair, one is for starter strip, and one for hip ridge. All colors of the same style will use the same starter strip. But each color will be mapped to a different hip ridge sku, as hip ridge are on the outside, and they have to match the same color as the shingle. See how to add hip ridge in next section.

  4. In itemAttributes.properties > sepecify image path and qty > also put image to the correct path
  5. roofingDefaultSku.properties - not sure if this part relavant gets used anymore as prices are dynamic now
  6. tiles-def.xml - locate screen that product is suppose to be display in
  7. tiles-def.xml - Under screen section > Add product scode to the category - category adding is another topic
  8. tsShingles.properties from test folder. Unfortunately, we also have to use duplicate codes for testing
  9. itemAttributes.properties from test folder. Unfortunately, we also have to use duplicate codes for testing
  10. Don't forget underlayment, check whether a new underlayment needs to be added or not, each roof style may use a different underlayment, see roofUnderlayment.properties and roofUnderlayment.java to control which underlayment should be used.

Adding new hip ridge

One shingle usually has one unique hip ridge for it.
Example: CastleBrook color GreenOil shingle usually will map to a CastleBrook GreenOil hipRidge.
One hip ridge also has a few properties, more than shingle

  1. Adding hipRidge is similar to adding new shingles
  2. create a new scode for hipridge in roofingSkuID.properties
  3. tsShingles.properies > fill properties
  4. The roofing shingle product should then map this scode for hipRidge

Adding underlayment

roofUnderlayment

Each roof style may need specific or default underlayment.properties. Define new underlayments here. See RoofUnderlayment.java to control which shingles get which underlayment. Use the if else, for special shingles that require certain underlayment.
