Design-It Config

A little frame work that handles our app environments. For example, if we wanted to point our app to stage, preveiw, or prod, we can change the configurations using the env setter tool or opening the design-it config files manually and changing the values in the file.

As of Nov2015+ we are calling designConfig from Reaf and will no longer put designit config dependency directly inside the app poms. but instead in the reaf pom.

Hints

Location: C:\menards\env

How to get A Design-It Config build version

Design-it Config is just like any other apps. You can pull it from the git repo and build it with maven or eclipse. When you build it, just extract the build from the target folder which will be in a zip file and extract the zip into the C:/menards/env/DesignItConfig directory.

  • Fetch/Pull the branch (Say DesignitConfig, and make sure in eclipse the target branch is opened.
  • In window explorer, browse to git folder or where that branch is.
  • Go into that branch/app folder one level
  • Right click on any folder and dos prompt here
  • type: mvn clean install
  • This will generate a target folder and zip files
  • You can either from here unzip zips or copy paste zip to deploy apps
  • Ex: for designit config, we just copy the stage.ua.campus.zip and extract it to our env under c: drive
