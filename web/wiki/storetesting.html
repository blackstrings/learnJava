Store Testing

  • Before going to the store, spend 10-15 minutes on the tests you will have to be testing or is assigned to test at the store
  • Team lead will provide in the email the people who will be going to the store and the apps each person will be responsbile fore
  • Go to redmine, click on the filter query for the app you will be testing ex: Q1R1 Landscaping
  • This should bring up all the tests you will need to test for the landscaping app
  • Go through each test and print directions somehow so you know what to test for

Tips while at the store

  • Test with data changes will not show up, take note of which tickets have data changes
  • If landscaping has a ticket with data change, that product won't show up, but check so app doesn't crash at least
  • For C Apps, you'll have to kill and restart memcache and memcache updater

Updating the app before testing

  • Ensure that the new data has been pushed to the store, not too sure how to check this
  • f6 > password > go to symantec
  • locate and find designit-config-2015 or 2016 - update that one look for success
  • check the env to see if the correct settings are set

  • Then look for the designit package - update that one which should download all the apps (garage, fence, deck)
  • Make sure you see the app you are testing otherwise the installer wasn't created - call team lead to check
  • If the download hangs in the progress bar too long, it could be the download isn't going or is too slow
  • check to see if the download is downloading from campus or the site server or if the installer is done uploaded to the site server
  • call jansen - desktop support for resolving this issue
  • Go down the list of each app and update so that each version is up to date, to know that the app is most recent
  • One trick is to look at the date, it should be very close to today's date
