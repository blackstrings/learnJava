JBOSS

A J2EE (Java 2 Enterprise Edition) container that comes with either TomCat or Jelly embedded for handling HTTP request. It provides an engine for our JSP (Java Server Pages) web pages. In other words, it is used to compile our JSP dynamically on the server and turn them into HTML web pages which then sends the data to the client side. FYI: You can't just run JSP pages on a regular web server. JSP is similar to ASP and PHP. In short, using JSP, it allows you to put programming codes inside your html to generate dynamic web pages.

Highlights

  • Typical workflow: webserver > Tomcat > JBoss > database
  • Used for internal applications
  • Siding & Roofing, and Menards Virtual Kitchen Design-it (MVKD) uses this
  • A server shared between intranet teams

Installing JBoss

  • Copy JBoss folder from F:/DesignIt/JBoss/jboss-5.1.0.GA to you local computer C:/JBoss
  • Add these two folder (RoofingEstimator.war) & (SidingEstimator.war) to C:/JBoss/jboss-5.1.0.GA/server/default/deploy
  • Create a shortcut of run.bat to your desktop - the file is located under C:/JBoss/jboss-5.1.0.GA/bin/
  • To run, just double click on the run.bat
  • Whatever gets created in siding or roofing will be located under the two folders you created

Competitor

  • Apache - a regular webserver that handles HTTP traffic
  • TomCat - a servlet/JSP engine that supports JSP development. Can also sit next to another webserver to allow asp/php/perl/cgi requests

MCR PCI Patch Update Test

When there is a MCR for PCI Patch, it means we have to perform a quick test on Siding, Roofing, and Menards Virtual Kitchen Design-it app

  • To start test go to site here
  • Run each test below for Siding, Roofing, and MVKD
  • If test is under alpha or beta, click on LB under Dev under each app
  • Remember for siding & roofing, you need to add something like this to the end of the url: =restart&selection(storeNumber)=3011 ...
  • For siding test while making sure everything worked, make sure you can save & recall your design
  • For roofing, check that you can print okay (usually 3 minute wait for printing to start printing so give it some time)
  • For kitchen, just check printing and select as many as you can
  • Overall check if images are missing and that the app starts
  • At end of testing 3 apps, assign MCR back to Merica, log under construction, status to "wait for beta stage" or the next stage

Hints and Hacks to Preview Changes

For CSS Changes

if you are only making css changes, you can preview the changes without having to start stop jboss server. It only works on chrome and firefox. As long as you go to a new page and return you should see the changes. If this method doesn't work, you can use the below method which is slightly longer without re-deploying. If going to a new page and back doesn't preview your changes, you'll want to try clearing all your cache history.

You can even work on the css directly with firefox firebug, get the setting you like, then copy paste the css to the actual code file

    Works with the following:
  • Firefox - works with css changes only - go to new screen and back to see the change
  • Chrome - works with css changes only - you'll have to refresh the browser (CTRL+F) to see the css changes
  • IE - You may not have to stop/start server to see the changes - Try refresh Page first so the css cache is clear, if you don't see the changes, then you'll have to start/stop server
