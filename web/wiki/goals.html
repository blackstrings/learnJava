Test-Driven Developemtn (TDD)

  • The idea in TDD is to implement code that makes the product work and no more.
  • Also Test-driven development is a development technique where you must first write a test that fails before you write new functional code.
  • Take small steps
  • TDD Rhythm
    1. 1. Quickly add a test
    2. 2. Run all tests and see the new one fail
    3. 3. Make a little change
    4. 4. Run all tests and see them all succeed
    5. 5. Refactor to remove duplication
