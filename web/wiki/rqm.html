RQM

A third party online site we use to perform regression test. Regression testing is a way for us to test so that our codes are progressing and not regressing. In other words, our codes are not moving backward but forward.

How it work is that the test cases created from the last code release will eventually be used to test the app (that has the new code changes) AKA regression testing. This is to see that the same functions still works on the updated app.

In other words, we use the previous test cases, even though our codes may have changed, to test the latest release to ensure that our latest release still has the same functionality as the last release. Again, the functionality should be the same, though the codes may have changed.

In order to test in RQM, you will need someone to create a test script, put inside a test case, which is put inside a test suite and then have someone run the suite. The actual testing process will have the tester person go down a list of test steps, which is under a test script, one by one as you check yes and no if a certain test passes or not.

Do not email every fail test cases to the primary. The primary person will know what failed when reviewing your test cases you have done for them.

You will use the computer in the lab to do the testing (usually the beast) as we want to simulate a kiosk that is at a store. You will use one screen for RQM while the other kiosk screen to perform your test.

Setup

  • You will need an account to login, this login is separate from all your other accounts
  • When it comes to regression testing, you use one of the computes in the lab, usually the beast to run the regression test
  • Make sure the kiosk is pointing to the right store in order for the right images to show up, ask for help if this needs to be done

Technical Instructions to Test Suites, Cases, and Scripts

  • Lab computer > Login using myMenards > IS Tools > Rational Quality Management (RQM) > Login
  • Query your search if you know what test to perform, otherwise you'll see a bunch of test suites
  • select the appropriate test options and always create a new test sheet so you don't override the old test done by others
  • To pause, there is a little setup you need to do first by hitting the pause button
  • If you stop without finishing the test suite completely, all data of testing will be lost, so either pause it or finish the complete suite to ensure the testing is saved

Creating Test Suite, Test Cases, Test Scripts

  • This is the hierarchy of how test gets created.
  • 1 Suite can only have 1 Case, and 1 Case can only have 1 Script
  • 1 Script can have multiple steps
  • Recommend creating Test script first, then case, then suite
