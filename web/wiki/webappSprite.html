WebApps Converting to sprites

There are multiple property places that need to be in place for a main roofing style shingle products.

Roofing Siding Sprites look at a glance

Sprite Sample

A better sprite would not have white spaces inbetween. Gather images that need to be under one sprite. In addition, the size of the sprite doesn't matter, but try to keep a square layout and in powers of 2. 256x256, 512x512, etc.

However, in our webapps the different sizes of our buttons makes this rule hard to follow. So the sizes are random and are not in powers of 2.

tiles-def.xml

tiles-def.xml - as you can see only the layout needs to be change or target the correct layout. The scodes stays uneffected.

ItemBroker.Java

ItemBroker.java - the middle man between java and the jsp page. It handles retrieving data from the appContext and sets the data into the item object. make sure to add this to properly map each item to its sprite class. Put it in both methods getItem and getItemInfo

Item.Java

DsiplayItem.java - ensure this bean has the property spriteImageCssClass and a getter and setter. It is important to append prefix method name with get and set. The item broker will use these methods to set the values later at runtime.

CSS

mainStyles.css or the corresponding css, create a new unique class name per image. They each call the same sprite image, just with a different position per image. The size and width should stay the same however as this is the crop size area.

layout.jsp - set image class
