Garage Selections List

This page provides some documents on selection list in garage

Knowing if selection List has item

selections->pWCG1
using the selections-> pointer you can name the item "pWCG1" and it will return false or true. There may be no setup needed. "pWCG1" has to be a valid name in order for the function call to work

// means if selection list don't have wainscot for all four wallls
// without ! means selectionList has
!selections->pWCG1 && !selections->pWCG2 && !selections->pWCE1 && !selections->pWCE2

Accessing selection ID's values in INI

You need to set up 3 things

First one gets you the value from the INI in a string format. The itemID is what determines what ini and tag to look under

// get value by item id
GetStringINI(temp->ServiceDoorID,"DoorType","",szDoorType,sizeof(szDoorType),PathStrings::getWallIni1());

// get value by just tag name inside INI
GetIntINI(entry, "qty", 0, PathStrings::getHeaderIni());

Then convert the string to the correct type

// convert string to double
double windowWidth= atof(width);

further gets

// get the Scode sku (headerSkuID holds a string of the tagName inside INI)
// "sku" is the nodeName under the tagName within the INI
CUtil::GppsIniValue(headerSkuID, "sku", PathStrings::getGarageSkuIni1(), headerSku, sizeof(headerSku), LOCATION);
