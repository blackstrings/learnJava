Deck

This is a Java application that allows users to create beautiful residential decks that are either near their house, attached to their house, or stand alone. The best way to learn Deck is to learn the products themselves.

Hint

  • If you need to modify or change something on the java app, ask yourself if this is a change on a certain screen, if so start by looking for the correct XML screen, from there it may be the center source to get to all other functions and screens
  • Material List - useful for testing purposes

Editing & Debugging

When editing with eclipse, You should be able to see your code or property changes when you hit next or back and come back to the same screen. If not, your eclipse is not set to build automatically. In eclipse go to Project > set to "Build Automatically" if not already set.

Unique features the app supports

  • Different deck levels, ramps, railings, etc
  • Inserting pool or hot tub (not estimated though)

Setup

If you get a window pop up that DeckEstimator cannot be started error, make sure you name your project in eclipse "DeckEstimator"

Additional Screens and ScreenFlow Info

When running an app in debug mode, the blue highlighted xml are the inner panels called in that particular main screen. Use those to locate the panels to make modifications.

*screen.XML

    <init>

  • init - show/hide grab spots - controls what selections can be made
  • determine if current screen has to be skipped
  • set local XML variable s used for screen
  • load vital info for screen

    • <text>

  • Location - position on screen/panel
  • Size - text box size max length limit before new line starts
  • Align = similar to word doc alignment
  • Font = Reaf settings for font sizes/colors, only fonts specified in fonts.LUA
  • Message = what the text say

    • <display>

  • used to include script before making the message, to input dynamic text
  • can also hide/show certain images
  • convert variables that are not strings into proper readable variables

    • <fixedPanel>

  • Each fixedPanel has its own ideology and functions

    • <autoPanel>

  • Most of the product selections use autoPanel because they get their information from databases. To add or remove buttons, we simply change the database and the button will be removed or added.
  • Can set columns and rows, and next and prev buttons will appear only if the number of items loaded exceed the col and rows.

    <image>

  • Can be auto loaded into auto panel
  • Can be loaded into fixed panel

Save and Recall

We can save designs and recall them in deck. One tricky thing about recall is that if you are in prod environment, and you save, you have to change the environment to stage and then recall the design. If you are in stage, and save, you shouldn't have to change the environment and should be able to recall the design while still under the same stage environmnet.
